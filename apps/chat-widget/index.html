Chat Widget Development

Development Mode

This page demonstrates the chat widget in development mode. By default the widget auto-initializes when the data-chat-widget attribute is present, but in this demo we disable auto init with data-auto-init="false" so we can demonstrate custom styling via host-defined CSS variables.

Environment

Make sure the API server is running on port 3001 (or the port specified in your .env file).

Quick Start

# Start the API server
cd apps/api
pnpm dev

# In another terminal, start the widget dev server
cd apps/chat-widget
pnpm dev

Usage Examples

Declarative HTML (auto-init):

<div data-chat-widget
     data-api-url="http://localhost:3001"
     data-conversation-id="optional-id">
</div>

Manual init with CSS variables:

<style>
  #my-widget {
    --jexity-assistant-bg-chat-message-user: #ff0000;
    --jexity-assistant-text-chat-message-user: #ffffff;
  }
</style>

<div id="my-widget" data-chat-widget data-auto-init="false"></div>

<script type="module">
  import { initChatWidget } from '/src/main.tsx';
  import '/src/styles.css';

  initChatWidget({
    containerElement: document.querySelector('#my-widget'),
  });
</script>
